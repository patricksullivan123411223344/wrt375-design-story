Can LLM NPCs feel real?

Project Abstract

This project explores how AI-driven non-player characters can create more immersive and human-like interactions in videogames. Traditional NPCs often feel static, predictable, and emotionally disconnected becausde they do not remember past interactions or adapt meaningfully to player behavior.

To investigate this problem, I built a prototype NPC conversation system that reacts to user tone, tracks emotional state changes, and attempts to recall previous moments in the interaction. Participants tested the prototype through structured scenarios including first contact, hostile interaction, reconciliation, and memory recall.

User testing showed that players are not simply looking for the most advanced AI model. They are looking for an NPC that feels believable. Across both technical and non-technical testers, the main immersion breaks were response latency, hallucinated or drifting responses, and dialogue that felt too long or overly dramatic.

These findings led to a design pivot: instead of using the LLM to generate every line of dialogue in real time, the stronger approach is to use AI as a control layer that selects from authored dialogue, scripted events, and memory-aware responses. This keeps the experience adaptive while improving pacing, reliability, and immersion.

System Overview

The prototype was designed as a stateful NPC system rather than a simple chatbot. Instead of treating each message as an isolated prompt, the system carries structured information forward across the interaction so the NPC can respond with a consistent personality, emotional state, and memory of prior events.

At the center of the system are dataclass payloads that organize important context before it is sent into the model. These payloads help separate stable information, such as the NPC’s identity and personality, from changing information, such as the user’s tone, recent actions, emotional shifts, and conversation history.

The system also uses supporting instruction files to inject design rules and behavioral expectations into the conversation. These files help guide the NPC’s tone, response style, and role within the interaction, while memory systems track both short-term conversation context and longer-term relationship patterns.

This architecture allowed the NPC to behave more consistently than a standard one-off LLM response. The goal was not just to generate dialogue, but to simulate a character that could remember, react, and change based on the user’s behavior.
